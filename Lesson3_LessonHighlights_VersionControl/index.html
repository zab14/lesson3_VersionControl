Highlight #1 Virtual Control System

A VCS or version control system, automatically takes notes on who changes a file and also when they changed it. It can also merge changes to the same file. The benefits of a VCS are that the files within it cannot be overwritten. Also multiple files can be worked on by multiple people at the same time, because of a common repository.

Highlight #2 Git

Git is a distributed virtual control system. Git makes it so that every user of it has a complete copy of the repository on there computer. It also helps to prevent files from being overwritten as well. Their are three basic steps to Git and they are: Add, Commit, and Push.

Highlight #3 Using a Repository to Control History

Once a repository is created, the users can then start to record their changes within it. They can track their change history's and also start tracking their status as well. This is useful if the user wanted to check on what changes were made or see who made the changes.

Highlight #4 Remote Repository

A remote repository is a repository that resides on a remote server. Meaning that the repository can be shared among multiple users on a team. This is convinient becuase it allows all the team users to access the repository from any machine they are on.

Highlight #5 Local Repository

A local repository is different form a remote repository because it is not shared across multiple machines. It is only stored on a local machine of an individual user. Meaning that the user who has the local repository is the only one that can use it. You can create a local repository in two ways. One is to clone a currently existing repository and then transfer it over to your machine. The second method of creating a local repository is to just create a new repository.
